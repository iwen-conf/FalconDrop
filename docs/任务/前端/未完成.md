# 前端未完成任务

## 总目标

基于 `docs/rewrite-analysis/12-confirmed-web-scope.md`，实现纯 Web 前端：登录入口、照片页面、系统信息页面。前端页面文案除专业术语外全部使用中文。

## 技术栈与范围豁免

- 前端基线：React + TypeScript + Vite + Tailwind CSS + shadcn/ui + Zustand + TanStack Query + React Router + React Hook Form + Zod + Framer Motion。
- UI 风格：工具型 Web 管理界面，克制、信息清晰、适合现场操作；不做营销落地页。
- 配色：按 Lazycat 创建基线选择一个低饱和 shadcn CSS variables 方案，建议使用 Sage 或 Slate，避免大面积高饱和渐变。
- 表单：账号与 FTP 配置表单使用 React Hook Form + Zod。
- 服务端状态：REST 快照使用 TanStack Query。
- 局部 UI 状态：预览层、侧栏、WebSocket 连接状态等使用 Zustand 或组件本地状态。
- 动效：Framer Motion 只用于轻量状态转换，例如预览层、删除确认、列表插入；不做影响效率的装饰动画。
- Lazycat 通用基线默认包含注册、双 token 和无感刷新；本项目已确认只有唯一默认系统账号，前端只做登录和会话恢复，不做注册页。

## 交付里程碑

### M1 前端工程基线

- 初始化 React + TypeScript + Vite。
- 配置 Tailwind CSS、shadcn/ui、React Router、TanStack Query、Zustand、React Hook Form、Zod、Framer Motion。
- 建立 API client、错误模型、路由守卫和基础 layout。

### M2 登录与会话

- `/login` 可用。
- 刷新页面后通过 `/api/auth/me` 恢复登录态。
- 未登录访问 `/photos` 和 `/system` 会跳转登录页。

### M3 照片工作流

- `/photos` REST 快照、时间分组、预览、删除确认可用。
- WebSocket 新增、覆盖、删除事件能驱动 UI 更新。
- 断线重连后重新拉取 REST 快照。

### M4 系统信息与运维提示

- `/system` 展示系统版本、hash、系统时间、系统账号、FTP 账号、FTP 状态、PASV、存储状态。
- 支持 FTP 启停和账号编辑。
- 匿名 FTP、PASV、存储不可写有明确中文提示。

### M5 验收闭环

- typecheck、test、build 通过。
- 主路径 E2E 可复现：登录 -> 系统页启动 FTP -> 上传照片 -> 照片页实时出现 -> 删除。
