## P0 任务清单

- [ ] 初始化 Go module、HTTP router、日志、配置加载和优雅关闭。
- [ ] 编写 `.env.example`。
- [ ] 建立 PostgreSQL 连接和 migration 执行流程。
- [ ] 创建 `system_accounts`、`ftp_account`、`app_settings`、`media_assets`、`transfer_events` 表。
- [ ] 从环境变量初始化唯一默认系统账号和默认 FTP 账号。
- [ ] 实现系统账号登录、退出、`/api/auth/me` 和会话 middleware。
- [ ] 实现默认系统账号更新。
- [ ] 实现默认 FTP 账号、密码和匿名策略更新。
- [ ] 验证并选定 Go FTP server 库。
- [ ] 实现 FTP server 启动、停止、状态查询和幂等控制。
- [ ] 实现 FTP 默认账号认证和匿名访问。
- [ ] 实现 PASV 端口范围和 public host 配置。
- [ ] 实现本地数据卷 storage backend 和路径穿越防护。
- [ ] 实现上传完成 hook。
- [ ] 实现文件 hash 计算。
- [ ] 实现同名同 hash 覆盖。
- [ ] 实现同名不同 hash 保留两份内部文件。
- [ ] 实现 MIME 检测、照片识别和 EXIF 时间解析。
- [ ] 实现无 EXIF 时间的兜底时间字段。
- [ ] 上传任意格式文件后写入 `media_assets` 和 `transfer_events`。
- [ ] 实现 `/api/photos`、`/api/photos/{id}`、`/api/photos/{id}/content`、`DELETE /api/photos/{id}`。
- [ ] 实现 `/api/assets` 和 `/api/assets/{id}`。
- [ ] 实现 `/api/system/info` 聚合系统信息。
- [ ] 实现 WebSocket hub 和事件广播。
- [ ] 发布 `system-status`、`ftp-started`、`ftp-stopped`、`asset-uploaded`、`asset-overwritten`、`photo-added`、`photo-deleted` 事件。
- [ ] 编写 Dockerfile 和 docker-compose。
- [ ] 配置 PostgreSQL、本地数据卷、FTP 控制端口和 PASV 端口映射。
- [ ] 编写认证、FTP、存储、媒体、WebSocket 和 Docker smoke tests。
- [ ] 通过 `go test ./...`、`go vet ./...` 和 Compose 配置校验。

- [ ] 不做 RBAC、多用户、审计日志。
- [ ] 不做 AI 修图和 AI 队列。
- [ ] 不做 NAS/S3/MinIO/对象存储。
- [ ] 不做 Android MediaStore、Windows Shell、Tauri IPC。
